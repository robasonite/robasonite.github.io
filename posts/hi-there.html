A bit about me

On this site and GitHub (maybe elsewhere, not sure yet), I go by the moniker RJK.

I am a GNU+Linux user and open-source enthusiast who is currently not employed in the tech sector. I would like to enter the tech sector, but it's currently becoming more anti-human with each passing day. So this page is less of a "cover letter" and more of a "let's see where this thing goes" sort of thing. In fact, the whole site is like that.

I consider myself fairly comfortable with Unix-like shells and know how to use several commands, but I wouldn't say I'm a power user. I've also played with several programming languages (Lua, Forth, C, a few Lisp/Scheme implementations, etc), but I don't really consider myself a developer. More often than not, I glue stuff together when I want to solve a problem or because I'm bored. Python is the language that I typically use for scripting, usually whenever I need to work with nested data structures.

I first started using Linux in high school, around the time of Windows 2000 and XP. Like many people, I grew up with Windows in the era of browser toolbars, Limewire, and AOL CDs. I shared a computer with the rest of my family and frequently had to unclog the system every time I wanted to use it. Spyware, pop-ups, fake antivirus junk, and toolbars kept me from doing homework assignments and playing games.

After a while, I got tired of it. After complaining about it in a Yahoo chat room, a few strangers mentioned "linux live cds". That's when I started playing with KNOPPIX, Puppy Linux, and various derivatives. By the time Windows 8 came out, I had graduated to using Linux (specifically Ubuntu) as a daily driver.

These days, the first thing I do when I get a new computer is replace Windows. The second thing I do is disable the Secure Boot crap. If the BIOS supports it, I will also switch on "legacy boot" and turn off UEFI.
